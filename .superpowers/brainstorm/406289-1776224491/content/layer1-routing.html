Layer 1, Section 4: Routing & Layout

Map-centric desktop interface with sliding panels

Design Principle: Map is Always Visible

NON-NEGOTIABLE: The Map Never Disappears
❌ DON'T DO THIS
  • Full-screen modals that cover the map
  • Navigation to separate "pages"
  • Pop-ups that block map interaction
  • Multi-page routes (logout → dashboard → reports)
✅ DO THIS INSTEAD
  • Sliding panels from right/left/top/bottom
  • Map remains visible behind panels
  • Click outside panel → panel slides away
  • Single-page application with overlay UI
💡 Why: Municipal admins need constant situational awareness. During disasters, seeing the map while processing reports is critical. The map is the command center's radar screen.

  • Numbers update in real-time

3. Context Panels (Slide In/Out)

Rich details without blocking the map

  • Incident Details Panel - Slide from right when pin clicked
  • Responder Status Panel - Slide from right when responder clicked
  • Pending Queue Panel - Slide from left when "Pending" clicked
  • Active Incidents Dashboard - Slide from left when "Active" clicked
  • Mass Alert Tools - Overlay panel (centered, semi-transparent)
Interaction: Click outside → panel slides away. ESC key → close all panels.

Keyboard Shortcuts

Power User Shortcuts for Surge Events
Map Navigation
  • ↑ ↓ ← → - Pan map
  • + / - - Zoom in/out
  • Home - Reset to municipality view
  • O - Toggle overlays panel
  • Esc - Close current panel
Quick Actions
  • V - Open pending queue
  • D - Open active incidents
  • R - Open responder dashboard
  • A - Open mass alert tools
  • Enter - Open selected item
💡 Critical during surges: When processing 50+ reports/hour, keyboard shortcuts are faster than mouse clicks. Every second saved matters.
