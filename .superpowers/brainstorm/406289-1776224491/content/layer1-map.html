Map Foundation Extensions

1. Municipal Boundaries Layer

Feature: GeoJSON or Firestore-based boundaries

  • Display all 12 municipalities as boundary lines
  • Admin's municipality highlighted in green
  • Adjacent municipalities in gray
  • Boundary labels on hover/click
  • Toggle visibility with layer control
Data Source: GeoJSON from municipalities collection or static file

2. Admin-Specific Map Controls

Enhanced controls for municipal admin operations

  • Layer toggles: incidents, responders, boundaries, heat map
  • Quick filters: active only, pending only, all incidents
  • Zoom to municipality button
  • Reset view (home) button
  • Admin mode indicator (top bar)
UX: Always visible, don't obscure map center

3. Dynamic Layer System

Pluggable layer architecture for future features

  • MapLayer interface for all layers
  • Base layers: OpenStreetMap (default), Satellite (optional)
  • Overlay layers: incidents, responders, boundaries, heat map
  • Each layer has: toggle, opacity, z-index
  • Add/remove layers without refactoring
Benefit: Easy to add new layers (weather, evacuation zones, etc.)

4. Performance for 50+ Markers

Optimize rendering for surge conditions

  • Marker clustering (group nearby pins)
  • Virtual rendering (only visible markers)
  • Debounced updates (don't re-render on every change)
  • Web Workers for geospatial calculations
  • Progressive loading (critical path first)
Goal: Smooth UI even with 100+ incident pins

Data Requirements

✅ What We Need

  • Municipal boundaries GeoJSON - All 12 municipalities with boundaries
  • Firestore collection - municipalities/{id} with GeoJSON data
  • Boundary styling - Green for admin's municipality, gray for others
  • Layer configuration - Which layers show by default, z-index values

⚠️ Implementation Notes

  • GeoJSON files can be large - consider caching or lazy loading
  • Boundary resolution: detail vs performance trade-off
  • Need to handle missing municipality data gracefully
  • Layer state must persist across panel open/close
