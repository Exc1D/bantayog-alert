Alert Creation Scope by Role

Who can create alerts and for whom?

✓

This is Correct

Municipal Admins → Their municipality only. Provincial Superadmins → Entire province. Citizens → Cannot create alerts (only reports).

Alert Creation Scope Matrix

Practical Example

Scenario: Flash flood in Barangay San Jose, Daet
Municipal Admin (Daet)
Creates alert → Targets: All citizens with registered location = Daet
Provincial Superadmin
Creates alert → Targets: ALL citizens in Camarines Norte (all municipalities)
Municipal Admin (Labo)
CANNOT create alert for Daet - out of scope

Location-Based Alert Filtering

Citizens only receive alerts for their registered municipality. If they're anonymous, alerts are based on current GPS location.

// Alert targeting logic
// For registered users:
alerts.where('targetMunicipalities', '==', user.municipality)

// For anonymous users:
alerts.where('targetMunicipalities', array-contains(gps.municipality))

Why This Scope Matters

  • Prevents alert spam - Labo admin can't flood Daet citizens with irrelevant alerts
  • Relevance - Users only see warnings that affect their area
  • Authority - Only official sources can broadcast alerts (verified accounts)
  • Privacy - Location-based targeting for anonymous users respects privacy
