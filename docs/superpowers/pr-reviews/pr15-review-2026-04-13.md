### 2. Missing `userPhone` prop for phone-linked anonymous reports (confidence: 88%)

**File:** `src/features/profile/components/RegisteredProfile.tsx` line 170

**Finding:** `MyReportsList` accepts an optional `userPhone` prop to fetch phone-linked anonymous reports. `RegisteredProfile` renders it with only `userId` — no `userPhone`. This means anonymous reports linked via phone will not appear in the "Your Reports" tab.

```tsx
// Current (line 170):
<MyReportsList userId={user.uid} />

// Should be:
<MyReportsList userId={user.uid} userPhone={/* extract from user profile */} />
```

**Fix:** Extract the user's phone from the auth user (or user profile document) and pass it to `MyReportsList`. If the phone is not yet available in `RegisteredProfile`, document the limitation explicitly.
